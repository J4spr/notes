# Story map

1. bepaal de reis
	-  bepaal de top doelen voor uw prodict 
	-  wat doet ons product?
	- Welk probleem probeert u op te lossen? 
2. Bouw de "Ruggengraat" van je verhaal 
	-  begin met de scnearios van je use cases
	-  niet te veel details
3. Activiteiten identificeren en groeperen
	-  must have stories 
	-  stories in kleinere taken opdelne
	- schiet ideen nog niet af 
4. uitsplitsen 
	-  verdeel de beschikbare over de user stories

# Use case diagram

- toont verschillende use cases
- mensen worden voorgesteld door stokmannetjes

## alle onderdelen uitgelegd
- Use case
	- ellips: duidt aan dat het om een use case gaat
	- tekst: use case naam + lijdend voorwerp
	- link: iedere use case moet gelinkt zijn aan een actor
- Actor
	- gebruiker van het systeem, kan een mens zijn of een ander systeem
	- linkerkant:  primary actor
	- rechter of onderkant: secondary actor
- aandachtspunten 
	- geen assumpties
	- wees volledig 
	- overerving nodig van de actoren maar...
